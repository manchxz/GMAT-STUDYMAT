What Table Analysis actually is

Table Analysis questions give you one sortable table — typically with 8 to 15 rows and 4 to 7 columns — followed by three sub-questions, each presented as a True / False / Cannot Be Determined statement. The table is interactive on the live exam: you can sort any column ascending or descending in one click. Sorting is the single most powerful move you have. Use it.

Table Analysis questions are not testing your statistical knowledge in depth. They are testing whether you can read a table accurately under time pressure. Most misses are caused by reading the wrong row, confusing columns, or sloppy mental arithmetic — not by lack of statistical training.

The Table Analysis core principles

  • Read column headers first. Know what each column means before reading any row.
  • Sort by the column the question cares about. Always.
  • Most "median", "max", "min" questions resolve in one sort.
  • Double-count nothing. Each row is one entity. Do not sum across rows that represent overlapping data.
  • "Cannot be determined" is real. If the table does not contain the data, that is the answer.

Sort discipline — your most powerful move

ELI5 · two simple examples

Example 1 · the bookshelf. If a librarian asks "which book is the thinnest?", you would not measure each one. You would line them up in order of thickness and look at the first one. Sorting a column does exactly that — it lines the rows up by one feature so you can read off the first or the last.

Example 2 · the race results. A list of runners with their finish times is given to you in order of bib number. To find the winner, you do not need to compute averages — you just sort by time and read the top row. Sorting by the right column is the entire trick.

Expert · which column to sort, by question type
  • Highest / lowest / max / min → sort that column descending or ascending. Read top or bottom row.
  • Median → sort that column. The middle row of the sorted set is the median.
  • Comparisons across two columns → sort by one column and visually scan the other.
  • "How many entities meet condition X" → sort or filter by the column in X, count.
  • Sum / mean → you usually need to compute. The table should give you a sum row at the bottom if it is the answer.

Statistical literacy — the five concepts you need

The minimum statistical toolkit

  • Mean sum of values ÷ count. Sensitive to outliers.
  • Median middle value when sorted. For even count, average of the two middles. Robust to outliers.
  • Mode most common value. Useful for categorical data.
  • Range max − min. A simple spread measure.
  • Standard deviation on the GMAT, "high SD" = spread out; "low SD" = clustered. Rarely a numeric calculation.
ELI5 · two simple examples

Example 1 · the test scores. Five students score 60, 70, 80, 90, 100. The mean is 80 (sum 400, divide 5). The median is 80 (middle value). The range is 40 (100 − 60). When mean equals median, the data is roughly symmetric. When they differ, the data is skewed.

Example 2 · the dinner check. Four people order meals costing $12, $14, $15, $99. The mean is $35 — but no one paid close to that. The median is $14.50, which actually represents what most people paid. Whenever one number is way bigger than the rest, the median is the more honest measure. The GMAT loves this contrast.

The T/F grid — how to execute three sub-questions efficiently

Expert · the parallel-eval routine

Each Table Analysis prompt usually shows three sub-statements at once. Resist the urge to evaluate all three on the same sort. Instead:

  1. Read all three sub-statements first. Identify which columns each one needs.
  2. Sort once per sub-statement (or recycle a sort if two sub-statements use the same column).
  3. Mark each as T/F/CBD before moving on. Do not leave any blank.
  4. Be skeptical of statements that "feel right". The wrong answer is often the one the table seems to suggest at a glance.

Table-specific traps

Trap · category overlap

If a table has a "category" column where one entity can belong to multiple categories (e.g., a product tagged "Electronics, Home"), do not sum the categories — you will double-count. Always check whether categorization is exclusive.

Trap · totals row hidden in the data

Some tables include a "Total" row at the bottom. If you sort by a numeric column, the total row may end up at the top or bottom and skew your max / min reading. Always check whether the row is an entity or a summary.

Trap · units across columns

One column may be in thousands, another in raw counts, another in percentages. Read the header carefully — including the parenthetical (e.g., "Revenue ($M)" vs. "Customer Count"). See Units Trap.

Explanations

Explanation · 01↑ back to Q01
Answer · True

Sort the Year column descending. Dogwood is at the top with $860K. True.

Explanation · 02↑ back to Q02
Answer · True

Sort Q4 ascending: 100, 110, 115, 130, 155, 170, 180, 230. With 8 values, median is the average of the 4th and 5th: (130 + 155) / 2 = 142.5. 142.5 > 130. True.

Explanation · 03↑ back to Q03
Answer · False

East has two stores. Elm goes 130 → 125 → 120 → 115 (decreasing ✓). Fir goes 75 → 85 → 95 → 100 (increasing ✗). Therefore not every East store had decreasing revenue. False.

Explanation · 04↑ back to Q04
Answer · C · $1.5M

Cedar 665 + Dogwood 860 = 1525 (in $K), or about $1.5M.

Explanation · 05↑ back to Q05
Answer · B · South

Sort the Year column descending: Dogwood ($860K) is at the top. Dogwood is in the South. So the highest single-store annual revenue is in the South.

Explanation · 06↑ back to Q06
Answer · True

Engineering salaries: 130, 170, 95 → sorted: 95, 130, 170 → median 130. Sales: 110, 75 → median 92.5. HR: 90, 70 → median 80. Marketing: 80, 95, 60 → median 80. Engineering's median (130) is the highest. True.

Explanation · 07↑ back to Q07
Answer · False

Filter to tenure > 7:

P-001: 8 yrs, $110K  → above $90K ✓
P-004: 10 yrs, $170K → above $90K ✓
P-006: 12 yrs, $90K  → exactly $90K, NOT above ✗

P-006 has tenure greater than 7 (12 years) but a salary of exactly $90K — which is not "above" $90K. The statement fails on this employee. False.

Structural takeaway: read every "above" / "at least" / "more than" boundary carefully. Strict (>) and non-strict (≥) are different.

Explanation · 08↑ back to Q08
Answer · True

Marketing reviews: 3, 4, 2 → median 3. Sales: 4, 3 → median 3.5. Engineering: 5, 4, 3 → median 4. HR: 5, 4 → median 4.5. Marketing's median (3) is the lowest. True.

Explanation · 09↑ back to Q09
Answer · C · $98K

Sum of salaries: 110 + 75 + 130 + 170 + 95 + 90 + 70 + 80 + 95 + 60 = 975. Mean: 975 / 10 = $97.5K. Closest choice: $98K.

P-001:  8 yrs, review 4   ✓
